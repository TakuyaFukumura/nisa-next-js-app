# NISA利用状況可視化機能 仕様書

**作成日**: 2026年3月2日
**対象リポジトリ**: nisa-next-js-app

---

## 1. 概要

NISAの利用状況をグラフで視覚的に表示するWebアプリケーションを実装する。  
データはデータベースを使用せず、リポジトリ内部に配置したCSVファイルから読み込む。

---

## 2. 削除対象（既存機能）

現在のアプリケーションはNext.jsのテンプレートリポジトリをベースにしており、NISA可視化とは無関係な機能が含まれている。以下を削除する。

### 2.1 削除対象ファイル・ディレクトリ

| 対象 | 理由 |
|------|------|
| `src/app/page.tsx` | テンプレートのホーム画面。NISAの画面に置き換える |
| `src/app/api/message/route.ts` | テンプレートのAPIルート。DB連携のため不要 |
| `lib/database.ts` | SQLite連携ライブラリ。DB機能を削除するため不要 |
| `__tests__/lib/database.test.ts` | `database.ts` のテスト。削除対象のため不要 |

### 2.2 削除対象の依存パッケージ

| パッケージ | 理由 |
|------------|------|
| `better-sqlite3` | SQLite接続ライブラリ。DB機能を削除するため不要 |
| `@types/better-sqlite3` | 上記の型定義。合わせて削除 |

## 3. 新規追加機能

### 3.1 画面一覧

| 画面名 | パス | 説明 |
|--------|------|------|
| NISA全体利用状況画面 | `/` | NISA全体の枠（1,800万円）に対する利用状況をグラフ表示 |

## 4. データ設計

### 4.1 CSVファイルの配置場所

```
data/
  nisa.csv
```

リポジトリ内の `data/` ディレクトリにCSVファイルを配置する。  
現状の `.gitignore` では `/data/` ディレクトリ全体が除外されているため、`data/nisa.csv` がGitで追跡されるよう、`.gitignore` に以下の例外ルールを追加する。

```gitignore
!data/nisa.csv
```

### 4.2 CSVファイルのフォーマット

#### `data/nisa.csv`

年ごとのNISA投資実績を記録するファイル。

```csv
year,tsumitate_amount,growth_amount
2024,400000,2400000
2025,400000,2400000
```

| カラム名 | 型 | 説明 |
|----------|----|------|
| `year` | 整数 | 投資年（西暦） |
| `tsumitate_amount` | 整数 | つみたて投資枠の利用額（円） |
| `growth_amount` | 整数 | 成長投資枠の利用額（円） |

### 4.3 NISA制度の定数

| 定数名 | 値 | 説明 |
|--------|----|------|
| NISA_TOTAL_LIMIT | 1,800万円 | NISA生涯投資枠上限 |
| TSUMITATE_YEARLY_LIMIT | 120万円 | つみたて投資枠 年間上限 |
| GROWTH_YEARLY_LIMIT | 240万円 | 成長投資枠 年間上限 |
| YEARLY_TOTAL_LIMIT | 360万円 | 年間投資枠合計上限 |

## 5. 画面設計

### 5.1 NISA全体利用状況画面（`/`）

NISA生涯投資枠（1,800万円）に対する現在の利用状況を表示する。

#### 表示内容

- **ページタイトル**: NISA利用状況
- **グラフ**: NISA全体枠の利用率を示すドーナッツグラフ
  - グラフ中央に利用率（%）を表示
  - 利用済みセグメント: 利用済み金額（円）をツールチップおよびラベルで表示
  - 残り枠セグメント: 残り枠金額（円）をツールチップおよびラベルで表示
- **サマリー情報**:
  - 生涯投資枠: 1,800万円
  - 利用済み合計: ○○円
  - 残り枠: ○○円
  - 利用率: ○○%
- **年別利用状況画面へのリンク**

### 5.2 年別利用状況画面（`/yearly`）

各年の年間投資枠（つみたて投資枠・成長投資枠）の利用状況を表示する。

#### 表示内容

- **ページタイトル**: 年別NISA利用状況
- **グラフ**: 年ごとの投資額を示す積み上げ棒グラフ
  - X軸: 年
  - Y軸: 投資金額（円）
  - 各棒グラフの内訳: つみたて投資枠（色A）・成長投資枠（色B）
  - 各年の年間上限（3,600,000円）を補助線として表示
- **テーブル**: 年ごとの詳細データ一覧
  - 年（年別内訳グラフ画面へのリンク付き）、つみたて投資枠利用額、成長投資枠利用額、合計、利用率

### 5.3 年別内訳グラフ画面（`/yearly/[year]`）

特定の年について、つみたて投資枠・成長投資枠それぞれの利用状況をドーナッツグラフで表示する。

#### 表示内容

- **ページタイトル**: ○○年 NISA内訳
- **グラフ（2つのドーナッツグラフを並列表示）**:
  - **つみたて投資枠グラフ**:
    - 中央に利用率（%）を表示
    - 利用済みセグメント: 利用済み金額（円）をラベル・ツールチップで表示
    - 残り枠セグメント: 残り枠金額（円）をラベル・ツールチップで表示
    - 年間上限: 120万円
  - **成長投資枠グラフ**:
    - 中央に利用率（%）を表示
    - 利用済みセグメント: 利用済み金額（円）をラベル・ツールチップで表示
    - 残り枠セグメント: 残り枠金額（円）をラベル・ツールチップで表示
    - 年間上限: 240万円
- **サマリー情報**:
  - つみたて投資枠: 利用済み ○○円 / 上限 120万円（○○%）
  - 成長投資枠: 利用済み ○○円 / 上限 240万円（○○%）

## 7. CSVデータ読み込み設計

### 7.1 `lib/csvLoader.ts` の仕様

サーバーサイドでCSVファイルを読み込み、型付きデータとして返す関数を提供する。

#### エクスポートする型・関数

```typescript
// CSVの1行を表す型
export type NisaRecord = {
  year: number;
  tsumitate_amount: number;
  growth_amount: number;
};

// CSVファイルを読み込んでNisaRecordの配列を返す
export function loadNisaData(): NisaRecord[];
```

### 7.2 データ取得の方式

CSVはサーバーサイド（Next.jsのServer Components）で読み込む。  
クライアントへは計算済みの集計データをServer ComponentsからのPropsとして渡す（本仕様の範囲ではAPIルートは使用しない。将来的に必要になった場合に検討する）。

## 8. グラフライブラリ選定

グラフの描画には **Recharts** を使用する。

| ライブラリ | バージョン | 採用理由 |
|------------|------------|----------|
| `recharts` | 2.x 系（例: 2.12.x） | React向けに設計されており、TypeScript対応が充実している。ドーナッツグラフ・棒グラフが容易に実装可能 |

## 10. 非機能要件

| 項目 | 内容 |
|------|------|
| レスポンシブ対応 | スマートフォン・タブレット・PCで表示崩れなく動作すること |
| ダークモード | 既存のダークモード切替機能に対応すること |
| TypeScript | 型定義を適切に行い、`any` の使用を避けること |
| テスト | CSVローダーおよびグラフコンポーネントのユニットテストを作成すること |
| Lint | `npm run lint` でエラーが出ないこと |
